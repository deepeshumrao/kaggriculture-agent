1. GOAL (official one-line description)
--------------------------------------------------------------------------------
"Put your course learning into action by building an agent that solves a
real-world problem, helps people or improves everyday living."

--------------------------------------------------------------------------------
2. CONTEXT — THE COURSE
--------------------------------------------------------------------------------
The capstone caps off the "5-Day AI Agents: Intensive Vibe Coding Course"
run by Google + Kaggle (course dates cited: June 15-19, 2026). "Vibe coding"
means using natural language as the primary programming interface to build
AI agents (rather than writing traditional code line-by-line).

Daily course structure:
  - Day 1: Introduction to Agents & Vibe Coding — natural-language workflows.
  - Day 2: Agent Tools & Interoperability — external APIs, code execution,
           agent-to-agent (A2A) communication.
  - Day 3: Context Engineering — sessions, skills, memory; long-term memory
           and state for personalized agents.
  - Day 4: Agent Quality & Security — testing, guardrails, reliability.
  - Day 5: Prototype -> Production — cloud deployment and observability.

--------------------------------------------------------------------------------
3. THE CAPSTONE — "KAGGRICULTURE"
--------------------------------------------------------------------------------
A hands-on farming SIMULATION challenge: build and deploy an autonomous agent
that competes on a LIVE LEADERBOARD. You design an agent that makes decisions
in the simulated farming environment; performance is scored competitively.

--------------------------------------------------------------------------------
4. DELIVERABLES / SUBMISSION REQUIREMENTS
--------------------------------------------------------------------------------
A submission is expected to include:
  1. A Kaggle Writeup documenting the agent (how it works, design choices).
  2. A video explanation of the agent.
  3. A brief rationale (why you built it the way you did).
  4. A link to the code (e.g., repo or Kaggle notebook).
  (Plus the deployed/submitted agent that runs on the live leaderboard.)
